# V1 Acceptance Checklist

Date: 2026-06-20

This checklist maps the first-version acceptance criteria to implementation and verification evidence. It is intended for thesis writing, defense preparation, and final project review.

## Summary

Backend evidence covers RBAC, 知识库 ingestion/search, RAG answer persistence, 工单 workflow, admin 统计 APIs, and Redis-backed queue design. Frontend evidence now covers the public homepage, login/session restore, demo guide, knowledge management, RAG chat with citations, ticket list/detail workflow, admin dashboard, and system role management. Remaining gaps are final demo corpus preparation and optional SSE/file-upload hardening depending on how strictly the final defense interprets those two acceptance items.
